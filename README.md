# Rocket Motor Design Optimizer

A physics-informed ML pipeline for optimizing solid rocket motor (SRM) designs. The system generates a synthetic dataset from parametric simulations, trains a neural surrogate model to approximate the physics, and runs NSGA-II multi-objective optimization to find Pareto-optimal designs across performance and safety constraints.

---

## Overview

Direct physics simulation is accurate but slow. Running thousands of evaluations inside an optimization loop makes it impractical for design-space exploration. This project replaces the simulation with a neural surrogate model, enabling fast multi-objective search while preserving physical fidelity.

The pipeline has four stages:

1. **Simulation and dataset generation** -- parametric BATES grain designs are sampled, evaluated against a physics model, and filtered for validity.
2. **Exploratory data analysis** -- correlation structure between design parameters and performance metrics is characterized before any model is trained.
3. **Surrogate-driven optimization** -- NSGA-II searches the design space using the surrogate as a fitness function, completing 20,000 evaluations in 3.48 seconds.
4. **Interactive UI** -- a Streamlit app exposes the trained surrogate and optimizer for interactive design exploration without touching the CLI.

---

## Dataset

The dataset (`bates_motor_dataset.csv`) was generated by simulating 10,000 parametric rocket motor designs (via OpenMotor). Designs that violated safety constraints were removed, leaving **8,922 valid entries**.

**Filtering criteria applied during generation:**
- `max_pressure_psi < 1000` (structural safety limit)
- `total_impulse_ns > 0` (physically valid combustion)

**Input features (design parameters):**
| Parameter | Description |
|---|---|
| `diameter` | Motor outer diameter (m) |
| `bates_len` | BATES grain segment length |
| `nozzle_throat_dia` | Nozzle throat diameter |
| `nozzle_convAngle` | Nozzle convergence half-angle |
| `nozzle_divAngle` | Nozzle divergence half-angle |

**Output metrics (performance targets):**
| Metric | Description |
|---|---|
| `total_impulse_ns` | Total impulse (N-s) |
| `max_pressure_psi` | Peak chamber pressure (PSI) |
| `port_throat_ratio` | Port-to-throat area ratio |
| `peak_massflux_ratio` | Peak propellant mass flux ratio |

**Summary statistics:**
- Mean total impulse: 81.44 N-s (range: 5.36 to 211.58)
- Mean max chamber pressure: 398.72 PSI
- No missing values, no duplicate rows in the final dataset.

## Key Findings from EDA

**Nozzle throat diameter vs. chamber pressure (-0.80 correlation)**

The strongest single relationship in the dataset. A smaller throat creates higher back-pressure in the combustion chamber. Motors with small-category throats averaged 627.32 PSI; large-category throats dropped pressure significantly. However, medium throats achieved the highest average total impulse (85.44 N-s), revealing a trade-off: the throat size that maximizes pressure is not the same as the one that maximizes impulse.

**Motor diameter vs. total impulse (+0.80 correlation)**

The single most reliable lever for increasing total impulse. The scatter plot of `diameter` vs. `total_impulse_ns` shows tight clustering around the regression line, confirming a strong linear relationship. More diameter means more propellant volume, which drives impulse proportionally.

**BATES grain length vs. total impulse (+0.44 correlation)**

A secondary but meaningful contributor to impulse. Combined with diameter, grain geometry accounts for the bulk of impulse variance.

**Nozzle angles (near 0.00 correlation with all outputs)**

`nozzle_convAngle` and `nozzle_divAngle` showed negligible correlation with any performance metric within the simulated ranges. These parameters were deprioritized in optimization.

## Optimization

The neural surrogate replaces direct simulation calls inside the NSGA-II loop.

**Performance:**
- 20,000 evaluations (100 generations, population size 200)
- Completed in **3.48 seconds**
- Equivalent direct simulation runtime: hours

**Objectives (multi-objective, Pareto front):**
- Maximize `total_impulse_ns`
- Minimize `max_pressure_psi` (safety margin)

## Setup

**Requirements:**
- Python 3.10+
- PyTorch
- Streamlit
- pandas, numpy, matplotlib, seaborn
- pymoo (NSGA-II implementation)
- scikit-learn

```bash
pip install -r requirements.txt
```

**Run the Streamlit UI:**
```bash
streamlit run app.py
```

**Run the optimizer directly:**
```bash
python optimize_v2.py
```

**Regenerate the dataset:**
```bash
python simulate_motor.py
```

## Results

The Pareto front produced by NSGA-II identifies designs that trade total impulse against chamber pressure margin. The surrogate model's dominant predictor alignment (+0.80 on motor diameter) matches the physics ground truth, confirming surrogate fidelity. All optimized designs satisfy the 1,000 PSI safety constraint.
